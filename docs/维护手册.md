# AlgoWiki 维护手册

适用对象：运维人员、项目负责人、长期维护者。

## 1. 维护目标

- 服务可访问
- 数据可恢复
- 发布可回滚
- 安全事件可追踪
- 媒体文件可控
- 真实密钥不进入 Git

## 2. 关键文件

| 路径 | 用途 |
| --- | --- |
| `deploy/.env.production` | 生产环境变量 |
| `deploy/.env.test` | 测试环境变量 |
| `docker-compose.server.yml` | 服务器 Compose |
| `deploy/server-update-from-registry.sh` | 镜像更新 |
| `deploy/server-verify.sh` | 巡检 |
| `deploy/server-db-backup.sh` | 数据库备份 |
| `deploy/nginx.algowiki.conf` | 生产 Nginx 示例 |

真实环境文件必须设置为 `600` 权限，并且不得提交到 Git。

## 3. 日常巡检

建议每天检查：

```bash
cd /srv/algowiki
./deploy/server-verify.sh
docker compose --env-file deploy/.env.production -f docker-compose.server.yml ps
docker compose --env-file deploy/.env.production -f docker-compose.server.yml logs --tail=100 web
```

健康检查：

```bash
curl -fsS <PUBLIC_SITE_URL>/api/health/
```

## 4. 日志

重点关注：

- 应用错误日志
- 安全审计日志
- 慢请求日志
- Nginx access / error log
- Docker container log

排查问题时优先记录：

- 时间
- 用户或请求路径
- `X-Request-ID`
- 当前 release
- 最近一次发布 commit

## 5. 数据库备份

建议至少保留：

- 发布前备份
- 每日自动备份
- 最近 7 天备份
- 每周归档备份

示例：

```bash
cd /srv/algowiki
./deploy/server-db-backup.sh --env-file deploy/.env.production
```

恢复前必须先：

1. 停止写入
2. 备份当前库
3. 确认恢复文件来源
4. 在测试环境验证

## 6. 媒体文件维护

当前低量级阶段可使用本地文件系统。需要关注：

- `MEDIA_ROOT` 磁盘空间
- 用户上传频率
- 图片审核结果
- 过期临时文件清理

清理命令：

```bash
docker compose --env-file deploy/.env.production -f docker-compose.server.yml exec -T web \
  python manage.py purge_expired_media --limit 1000
```

建议使用 cron + flock，避免并发清理。

## 7. 安全维护

必须保持：

- `DJANGO_DEBUG=0`
- 强 `DJANGO_SECRET_KEY`
- HTTPS
- `SESSION_COOKIE_SECURE=1`
- `CSRF_COOKIE_SECURE=1`
- 管理员最小化
- 定期检查封禁、举报、删除内容和安全日志

高风险操作：

- 修改用户角色
- 手动通过验证
- 删除或恢复内容
- 修改第三方服务密钥
- 数据库恢复

执行前应确认影响范围和回滚方案。

## 8. 发布与回滚

发布看 [发布与更新操作手册](./发布与更新操作手册.md)。

回滚原则：

- 优先回滚镜像
- 不随意手工改数据库
- 迁移失败先备份再处理
- 回滚后必须重新健康检查

## 9. 常见故障

| 现象 | 排查顺序 |
| --- | --- |
| 502 | Nginx upstream、容器端口、容器状态 |
| 登录失败 | 用户状态、Token、密码策略、时钟 |
| 上传失败 | 图片大小、审核配置、MEDIA_ROOT 权限、磁盘 |
| 页面空白 | 前端构建产物、Nginx fallback、浏览器控制台 |
| 审核不生效 | 后端日志、对象状态、权限、AI 审核配置 |
