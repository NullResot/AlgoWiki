# AlgoWiki 发布与更新操作手册

适用对象：负责测试发布、正式发布和回滚的维护者。

## 1. 发布原则

- 小步提交，小步发布
- 测试环境先验收，正式环境后发布
- 每次发布都要能定位 commit、镜像 tag 和 release 标识
- 生产环境禁止直接使用未验收分支
- 任何真实密钥只存在于服务器环境文件或平台 secrets

## 2. 标准流程

```bash
git checkout develop
git push origin develop

git checkout test
git merge --no-ff develop
git push origin test

git status  # 测试环境验收通过后
git checkout main
git merge --no-ff test
git push origin main
```

## 3. 测试环境更新

```bash
ssh <SERVER_SSH> "cd /srv/algowiki && ./deploy/server-update-from-registry.sh --env-file deploy/.env.test --sync-github-branch --github-branch test"
```

快部署：

```powershell
cd <project-root>
powershell -ExecutionPolicy Bypass -File .\scripts\quick-release-test-to-server.ps1 `
  -ServerHost <SERVER_IP_OR_DOMAIN>
```

## 4. 生产环境更新

```bash
ssh <SERVER_SSH> "cd /srv/algowiki && ./deploy/server-update-from-registry.sh --sync-github-branch --github-branch main"
```

更新脚本会：

- 同步目标分支最新提交
- 解析对应镜像 tag
- 拉取镜像
- 重建 Web 容器
- 执行迁移、收集静态文件和健康检查

## 5. 发布后验证

至少检查：

```bash
curl -fsS <PUBLIC_SITE_URL>/api/health/
```

再人工检查：

- 首页
- 登录
- 个人中心
- Wiki
- 问答
- 动态
- 审核台
- 管理后台
- 图片上传

## 6. 回滚

回滚到指定 commit 镜像：

```bash
cd /srv/algowiki
./deploy/server-update-from-registry.sh \
  --image ghcr.io/<owner>/algowiki-web:sha-<commit> \
  --release rollback-sha-<commit>
```

回滚后仍需执行完整健康检查和关键流程验证。

## 7. 发布异常处理

| 现象 | 处理 |
| --- | --- |
| Actions 没有构建镜像 | 检查分支、workflow 权限和 secrets |
| 服务器拉镜像失败 | 检查 registry、登录态和网络 |
| 健康检查失败 | 查看容器日志、数据库连接和环境变量 |
| 页面还是旧版本 | 检查浏览器缓存、CDN 缓存和镜像 tag |
| 迁移失败 | 停止发布，备份数据库，定位迁移文件 |
