# AlgoWiki 远程部署指南

适用对象：首次配置服务器、迁移服务器或远程维护部署环境的人员。

## 1. 推荐拓扑

```text
Local workstation
  |
  | SSH / SCP / GitHub Actions
  v
ECS / VPS
  |
  +-- Nginx
  +-- Docker Compose
  +-- Web container
  +-- MySQL / RDS
```

## 2. SSH 连接

本地 `~/.ssh/config` 示例：

```sshconfig
Host algowiki-server
    HostName <SERVER_IP>
    User <SERVER_USER>
    IdentityFile ~/.ssh/<PRIVATE_KEY>
    IdentitiesOnly yes
    ServerAliveInterval 60
```

远程项目目录建议固定为：

```text
/srv/algowiki
```

## 3. 服务器基线

首次进入服务器后：

```bash
cd /srv/algowiki
chmod +x deploy/server-bootstrap.sh
SWAP_MB=4096 INSTALL_NGINX=1 CONFIGURE_UFW=1 ./deploy/server-bootstrap.sh
```

该脚本负责：

- 安装 Docker
- 安装 Docker Compose Plugin
- 配置 swap
- 可选安装 Nginx
- 可选配置基础防火墙

## 4. 环境文件

生产环境从示例复制：

```bash
cp deploy/env.production.example deploy/.env.production
chmod 600 deploy/.env.production
```

必须修改：

- `DJANGO_SECRET_KEY`
- `DJANGO_ALLOWED_HOSTS`
- `DJANGO_CSRF_TRUSTED_ORIGINS`
- `DB_*`
- `SUPERADMIN_PASSWORD`
- 邮件、短信、图片审核等第三方服务配置

测试环境使用：

```bash
cp deploy/env.test.example deploy/.env.test
chmod 600 deploy/.env.test
```

## 5. 启动服务

注册表镜像方式：

```bash
cd /srv/algowiki
./deploy/server-update-from-registry.sh
```

本地镜像包方式：

```bash
cd /srv/algowiki
./deploy/server-compose-up.sh --env-file deploy/.env.production --image-archive /path/to/algowiki-web.tar
```

## 6. Nginx

生产模板：

- `deploy/nginx.algowiki.conf`

测试模板：

- `deploy/nginx.test.algowiki.conf`

应用配置后执行：

```bash
nginx -t
systemctl reload nginx
```

## 7. 验证

```bash
cd /srv/algowiki
./deploy/server-verify.sh
curl -fsS <PUBLIC_SITE_URL>/api/health/
docker compose --env-file deploy/.env.production -f docker-compose.server.yml ps
```

## 8. 常见问题

- 无法拉取镜像：检查 registry 登录、网络和镜像 tag
- 容器启动失败：查看 `docker compose logs --tail=200`
- 502：检查 Nginx 反代端口与 `APP_PORT`
- 数据库连接失败：检查 RDS 白名单、VPC、账号和密码
- 上传失败：检查 `MEDIA_ROOT` 权限和磁盘空间
