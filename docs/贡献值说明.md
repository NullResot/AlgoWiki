# AlgoWiki 贡献值说明

本文档说明 AlgoWiki 当前版本的贡献值口径。贡献值用于展示用户在内容建设、社区传播和高校协作中的参与情况，不等同于账号权限、身份认证或人工背书。

## 总体口径

当前贡献榜主要使用两类分值：

- 内容贡献值：来自 Trick 内容建设与 Trick 互动，系统字段为 `trick_contribution_score`。
- 社区贡献值：来自邀请新用户并完成验证，系统字段为 `invitation_score`。

综合贡献值按以下方式计算：

```text
综合贡献值 = 内容贡献值 + 社区贡献值
```

用户的评论、收藏、问题反馈、修订提交、公告操作等行为会记录为贡献事件或操作日志，但当前版本不直接计入贡献榜分值。

## 内容贡献值

内容贡献值主要围绕 Trick 内容产生，用于衡量用户对算法知识内容的建设与维护参与度。

当前规则如下：

| 行为 | 分值变化 | 说明 |
| --- | ---: | --- |
| Trick 通过审核 | +10 | 投稿通过审核后，作者获得内容贡献值。 |
| Trick 收到点赞 | +1 | 其他用户点赞后，Trick 作者获得内容贡献值。 |
| 取消点赞 | -1 | 点赞被取消后，作者对应收益回滚。 |
| 发起点踩 | -1 | 点踩者消耗内容贡献值。 |
| Trick 收到点踩 | -2 | 被点踩的 Trick 作者扣除内容贡献值。 |
| 取消点踩 | 点踩者 +1，作者 +2 | 点踩取消后，双方对应变化回滚。 |
| 删除审核确认有效 | 点踩者 +2 | 当 Trick 进入删除审核并由管理员确认删除时，参与有效点踩的用户获得奖励。 |
| 已审核通过的 Trick 被管理员删除 | -10 | 如果删除的是已通过审核的 Trick，作者原投稿奖励会回滚。 |

## 内容贡献值权限阈值

为了减少低成本刷互动和恶意点踩，部分 Trick 互动需要达到最低内容贡献值：

| 功能 | 当前阈值 |
| --- | ---: |
| 点赞 Trick | 内容贡献值达到 10 |
| 点踩 Trick | 内容贡献值达到 10 |
| 触发删除审核 | 同一 Trick 累计 5 个点踩 |

阈值只表示当前版本的系统配置，不代表永久规则。

## 社区贡献值

社区贡献值当前主要来自邀请系统，用于鼓励用户邀请真实的新用户参与 AlgoWiki。

当前规则如下：

| 行为 | 分值变化 | 说明 |
| --- | ---: | --- |
| 新用户使用邀请码注册 | 0 | 注册后先生成待生效邀请记录，不立即加分。 |
| 被邀请用户完成手机号验证 | +1 | 邀请记录变为有效后，邀请人获得社区贡献值。 |
| 邀请记录被回滚 | -1 | 已生效邀请被回滚时，邀请人的对应社区贡献值扣回。 |
| 邀请记录被驳回 | 视状态而定 | 未生效记录驳回不扣分，已生效记录驳回会先回滚对应奖励。 |

邀请奖励的核心条件是：被邀请用户需要完成手机号验证。单纯注册账号不会直接给邀请人增加贡献值。

## 邀请贡献回滚

以下情况可能导致已生效邀请被回滚：

- 被邀请用户被封禁。
- 被邀请用户注销账户。
- 被邀请用户被超级管理员彻底删除。
- 被邀请用户的手机号验证被撤销。
- 管理员在邀请管理中手动回滚或驳回邀请记录。

回滚只处理对应邀请记录产生的贡献值，不会影响邀请人通过其他方式获得的贡献值。

## 高校贡献值

高校贡献榜按用户填写的学校名称进行聚合。

当前计算方式如下：

```text
高校内容贡献值 = 该学校有效用户的内容贡献值之和
高校社区贡献值 = 该学校有效用户的社区贡献值之和
高校综合贡献值 = 高校内容贡献值 + 高校社区贡献值
```

未填写学校的用户不会进入高校贡献榜。已注销或已删除占位用户不会进入贡献榜统计。

## 近期贡献值

近期贡献榜用于展示最近一段时间内的活跃贡献情况。

当前口径如下：

- 统计周期：最近 30 天。
- 统计内容：最近 30 天内的内容贡献值变化和社区贡献值变化。
- 排序方式：先按近期贡献值排序，再按综合贡献值排序。

近期贡献值可能因为回滚事件出现负数或下降。

## 综合贡献榜

综合贡献榜展示用户的总体贡献情况。

当前排序口径如下：

1. 按综合贡献值从高到低排序。
2. 分值相同时，注册时间更早的用户优先。
3. 仍相同时，按用户 ID 排序。

综合贡献榜只统计有效用户，不统计已注销或系统占位账号。

## 活动记录与贡献值的区别

系统中存在普通贡献事件记录，例如：

- 收藏或点赞文章。
- 发表评论。
- 提交修订。
- 提交问题反馈。
- 发布动态。
- 管理员处理内容。
- 发布或管理公告。

这些记录主要用于个人中心、管理后台、审计和活动展示。当前版本中，它们不直接改变内容贡献值、社区贡献值或综合贡献值。

## 管理与审计

贡献值变化会保留事件记录，包括分值变化、触发用户、关联对象和回滚状态。管理员可以在后台查看邀请记录、邀请贡献事件和 Trick 贡献事件。

如果出现违规刷分、恶意互动或异常邀请，管理员可以通过封禁用户、撤销手机号验证、回滚邀请记录、删除违规内容等方式处理。系统会根据对应规则回滚相关贡献值。

## 规则调整

贡献值规则会根据社区运行情况调整。后续如果调整分值、阈值或统计范围，应同步更新本文档，并在必要时说明是否影响历史分值。
