# AlgoWiki 开发环境部署

适用对象：本地开发者、测试人员、二次开发者。

## 1. 前置条件

建议版本：

- Python 3.12+
- Node.js 22+
- MySQL 8.0+
- Git
- Windows PowerShell 或 Linux shell

SQLite 可用于快速验证，但正式开发建议使用 MySQL，以便尽早暴露字符集、迁移和事务问题。

## 2. 获取代码

```bash
git clone <REPO_URL> algowiki
cd algowiki
```

如果你已经在本机有仓库，直接进入项目根目录即可。

本文用 `<PROJECT_ROOT>` 表示你本机的项目根目录。它不是固定路径，取决于你把仓库 clone 到哪里。

`127.0.0.1` 表示只在本机访问，适合开发环境。不要为了公网访问直接把开发服务暴露出去，公网访问应使用 Nginx 或正式部署流程。

## 3. 后端环境

安装依赖：

```powershell
cd <PROJECT_ROOT>
venv\Scripts\python.exe -m pip install -r backend\requirements.txt
```

最小环境变量示例：

```dotenv
DB_ENGINE=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_NAME=algowiki
DB_USER=<DB_USER>
DB_PASSWORD=<DB_PASSWORD>
DJANGO_DEBUG=1
DJANGO_ALLOWED_HOSTS=127.0.0.1,localhost
```

本地 `.env` 不要提交到 Git。

## 4. 初始化数据库

PowerShell：

```powershell
cd <PROJECT_ROOT>
powershell -ExecutionPolicy Bypass -File .\scripts\init_database.ps1 `
  -DbEngine mysql `
  -DbHost 127.0.0.1 `
  -DbPort 3306 `
  -DbName algowiki `
  -DbUser <DB_USER> `
  -DbPassword "<DB_PASSWORD>" `
  -SuperadminPassword "<STRONG_PASSWORD>"
```

如只做本地临时验证，可改用 SQLite：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\init_database.ps1 `
  -DbPath ".\storage\db_live.sqlite3" `
  -SuperadminPassword "<STRONG_PASSWORD>"
```

## 5. 启动后端

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start_backend.ps1 `
  -DbEngine mysql `
  -DbHost 127.0.0.1 `
  -DbPort 3306 `
  -DbName algowiki `
  -DbUser <DB_USER> `
  -DbPassword "<DB_PASSWORD>" `
  -ServerHost 127.0.0.1 `
  -Port 8001 `
  -Background `
  -HealthCheck `
  -PidFile ".\storage\backend.pid"
```

健康检查：

```powershell
Invoke-WebRequest http://127.0.0.1:8001/api/health/ -UseBasicParsing
```

## 6. 启动前端

```powershell
cd <PROJECT_ROOT>\frontend
npm install
npm run dev
```

默认访问：

```text
http://127.0.0.1:5173
```

Vite 会把 `/api` 代理到后端。

## 7. 验证

后端测试：

```powershell
venv\Scripts\python.exe backend\manage.py test wiki.tests
```

前端构建：

```powershell
cd frontend
npm run build
```

## 8. 常见问题

- 数据看起来丢失：检查 `DB_ENGINE` 和 `SQLITE_NAME` 是否切到了另一个库
- 登录失败：检查数据库迁移、超级管理员密码和用户状态
- 前端请求失败：确认后端端口、Vite 代理和 CORS/CSRF 配置
- 图片上传失败：检查 `MEDIA_ROOT` 权限和图片安全配置
