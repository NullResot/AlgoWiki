# AlgoWiki 项目说明书

## 1. 项目定位

AlgoWiki 是一个面向算法竞赛/知识整理场景的内容协作平台，目标是提供：

- 结构化 Wiki 知识库
- 问答互动
- 修订审核
- 用户与权限管理
- 公告、通知、贡献记录
- 适合校园团队或小型社区的后台治理能力

项目当前采用前后端分离架构：

- 后端：Django + Django REST Framework
- 前端：Vue 3 + Vite + Pinia + Vue Router
- 数据库：MySQL 为主，SQLite 仅作临时兼容或测试用途

## 2. 总体架构

### 2.1 逻辑结构

1. 浏览器访问前端页面
2. 前端通过 `/api/*` 调用 Django REST API
3. Django 使用 ORM 访问 MySQL
4. 静态资源由 `frontend/dist` 构建后交由 Django/WhiteNoise 或 Nginx 反向代理提供
5. 上传文件保存到 `backend/media`
6. 审计日志、运行日志和运行期辅助文件位于 `storage/`

### 2.2 推荐生产部署结构

推荐的公网部署结构如下：

```text
Internet
  |
  v
Nginx / 云负载均衡
  |
  v
Gunicorn + Django
  |
  +-- frontend/dist
  +-- /api/*
  +-- /media/*
  |
  v
MySQL 8.x
```

单机部署也支持：

- Docker Compose：适合学生项目、演示站、低并发场景
- Gunicorn + Nginx：适合传统 Linux 服务器部署

### 4.1 根目录

- `backend/`：Django 后端

## 5. 主要角色

- 匿名用户：浏览公开内容
- 普通用户：注册、登录、评论、提问、提交修订
- 学校用户：在学校作用域分类具备内容审核/编辑能力
- 管理员：内容治理、用户治理、公告管理、审核与运维视角数据
- 超级管理员：拥有管理员全部权限，并可进行关键角色调整

## 7. 当前已完成的安全与稳定性设计

### 7.1 安全基线

- `DEBUG=0` 时强制要求显式 `SECRET_KEY`
- `ALLOWED_HOSTS`、`CSRF_TRUSTED_ORIGINS` 环境化
- 生产 HTTPS 强制跳转与 HSTS
- `HttpOnly`、`SameSite`、`Secure` Cookie 控制
- `X-Frame-Options: DENY`
- `X-Content-Type-Options: nosniff`
- `Referrer-Policy: same-origin`

### 7.2 认证与账号保护

- 登录/注册/改密限流
- 登录失败按用户名 + IP 窗口锁定
- Token TTL
- 改密后 Token 回收
- 封禁/软删除后 Token 回收

### 7.3 稳定性与可观测性

- `/api/health/` 健康检查
- 请求级 `X-Request-ID`
- 应用日志与安全日志分离
- 慢请求日志
- 容器启动前数据库等待
- `collectstatic`、部署检查、Gunicorn 部署支持

## 10. 健康检查与运行状态

`GET /api/health/` 返回以下核心信息：

- 数据库连接是否正常
- 媒体目录是否可写
- 剩余磁盘空间是否达到阈值
- 前端构建产物是否存在
- 当前安全关键配置状态
- 当前请求 ID

该接口适合接入：

- 手工巡检
- Nginx/云监控探针
- Docker 健康检查

## 11. 后续建议

如果项目要进入长期公网运行，建议后续继续推进：

1. 接入 Redis 做统一缓存与分布式限流

4. 补充分离式前端测试和关键流程 smoke test
5. 将超大后端视图与超大前端页面逐步拆分
