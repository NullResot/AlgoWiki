# AlgoWiki 运行命令与数据库说明

这是一份本地运行与数据库速查，不依赖固定盘符或固定克隆目录。

## 0. 进入项目根目录

后续命令都假设你已经进入仓库根目录，也就是能看到 `backend/`、`frontend/`、`scripts/`、`docs/` 的目录。

```powershell
cd <PROJECT_ROOT>
$ProjectRoot = (Get-Location).Path
$DbPath = Join-Path $ProjectRoot "storage\db_live.sqlite3"
```

`<PROJECT_ROOT>` 需要替换为你自己的项目目录。不要照抄别人的盘符或用户名路径。

## 1. 固定数据库路径

如果你使用 SQLite 做本地开发或应急验证，请始终使用同一个数据库文件：

```powershell
$env:DB_ENGINE = 'sqlite'
$env:SQLITE_NAME = $DbPath
```

只要路径不变、文件不删，文章、公告、问答、评论、工单、动态等数据就会保留。

## 2. 初始化数据库

首次部署或需要重建时执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\init_database.ps1 `
  -DbPath $DbPath `
  -SuperadminPassword "<STRONG_PASSWORD>" `
  -Rebuild
```

说明：

- `-Rebuild` 会清空旧库后重建
- 超级管理员账号默认由初始化脚本创建
- 密码请使用你自己的强密码，不要复用其他站点的密码

## 3. 启动后端

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start_backend.ps1 `
  -DbPath $DbPath `
  -ServerHost "127.0.0.1" `
  -Port 8001 `
  -Background `
  -HealthCheck `
  -PidFile ".\storage\backend.pid"
```

停止后端：

```powershell
if (Test-Path .\storage\backend.pid) {
  Stop-Process -Id (Get-Content .\storage\backend.pid) -Force
  Remove-Item .\storage\backend.pid -Force
}
```

## 4. 启动前端

```powershell
cd .\frontend
npm install
npm run dev
```

如果你需要继续执行后端相关命令，先回到项目根目录：

```powershell
cd $ProjectRoot
```

## 5. 健康检查

```powershell
Invoke-WebRequest http://127.0.0.1:8001/api/health/ -UseBasicParsing | Select-Object StatusCode
```

## 6. 备份与恢复

### 备份

```powershell
$BackupPath = Join-Path $ProjectRoot ("storage\backup_{0}.sqlite3" -f (Get-Date -Format yyyyMMdd_HHmmss))
Copy-Item $DbPath $BackupPath
```

### 恢复

```powershell
$BackupPath = Join-Path $ProjectRoot "storage\backup_YYYYMMDD_HHMMSS.sqlite3"
Copy-Item $BackupPath $DbPath -Force
```

## 7. 数据存储原则

数据库会保存这些核心数据：

- 用户、角色、权限
- Wiki 文章、分类、评论、修订
- 问答、收藏、通知
- 动态、图片、举报与审核记录
- 公告、工单、后台操作日志
- AI 助手配置与调用记录

不要删除数据库文件，除非你明确要重建环境。

## 8. 生产环境提醒

生产环境不要继续使用本地 SQLite。上线时应切换到 MySQL，并使用单独的环境文件、备份策略和运维手册。
